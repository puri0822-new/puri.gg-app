## 2. 배경 및 전환 사유

### 2.1 문제 정의

기존 puri.gg 웹 서비스는 Next.js 기반으로 데스크톱 브라우저에 최적화되어 있습니다. 그러나 내전 참여자의 실제 사용 패턴을 분석한 결과, **경기 중·직후 모바일 기기로 전적을 확인하는 빈도**가 압도적으로 높았습니다.

### 2.2 전환 근거

| 구분 | 웹 (현재) | 모바일 앱 (목표) |
|------|-----------|-----------------|
| 접근성 | 브라우저 URL 직접 입력 필요 | 홈 화면 아이콘 원터치 접근 |
| UX | 마우스 기반 인터랙션 | 터치·스와이프 네이티브 제스처 |
| 알림 | 미지원 | 경기 결과 Push Notification 가능 |
| 오프라인 | 미지원 | TanStack Query 캐시로 오프라인 조회 가능 |
| 성능 | 네트워크 의존 렌더링 | 네이티브 렌더링으로 60fps 보장 |

### 2.3 비즈니스 목표

- 일간 활성 사용자(DAU) 30% 이상 증가
- 전적 확인 세션당 체류 시간 2배 향상
- 경기 입력 소요 시간 단축 (웹 대비 40% 이내)

## 3. 핵심 기능 요구사항

### 3.1 소환사 검색 (F-01)

- **설명**: 닉네임 입력 시 실시간 자동완성으로 소환사 전적 페이지 이동
- **우선순위**: P0 (Must Have)
- **세부 요건**:
  - 입력 디바운스 300ms 적용
  - ELO 순 정렬된 자동완성 드롭다운 표시
  - 최근 검색 기록 로컬 저장 (최대 10개)
  - 검색 결과: 승률, KDA, 모스트 챔피언 3종 표시

### 3.2 ELO 리더보드 (F-02)

- **설명**: 전체 내전 참여자의 ELO 기반 순위표
- **우선순위**: P0 (Must Have)
- **세부 요건**:
  - 무한 스크롤(FlatList) 방식 랭킹 목록
  - TOP 3 카드형 하이라이트 UI
  - ELO / CP / 승률 / KDA 기준 정렬 탭
  - 티어 이미지(아이언~챌린저) 표시
  - Pull-to-Refresh로 실시간 갱신

### 3.3 전적 관리 (F-03)

- **설명**: 내전 경기 결과 입력 및 히스토리 조회
- **우선순위**: P0 (Must Have)
- **세부 요건**:
  - 블루팀 / 레드팀 5:5 구성 입력 폼
  - 챔피언 선택 (Data Dragon 이미지 연동)
  - KDA·포지션 입력
  - 관리자 권한 분리 (Firebase Auth UID 기반)
  - 입력 완료 시 Firestore 즉시 저장 및 ELO 재산출

### 3.4 소환사 상세 페이지 (F-04)

- **설명**: 개별 소환사의 누적 통계 대시보드
- **우선순위**: P1 (Should Have)
- **세부 요건**:
  - 티어 배지 + ELO 변화 그래프
  - 챔피언별 통계 (승률, KDA, 픽 수)
  - 최근 20경기 피드

### 3.5 통계 페이지 (F-05)

- **설명**: 전체 내전 메타 통계
- **우선순위**: P1 (Should Have)
- **세부 요건**:
  - Most Pick / 최고 승률 챔피언
  - 포지션별 분포 차트
  - 시즌 전체 요약 카드

## 4. 비기능 요구사항

| 항목 | 목표 |
|------|------|
| 초기 로드 | Firestore 첫 응답 2초 이내 |
| 렌더링 | 리스트 스크롤 60fps 유지 |
| 오프라인 | TanStack Query staleTime 5분, 캐시 유지 24시간 |
| 보안 | Firestore Security Rules로 읽기/쓰기 권한 분리 |
| 접근성 | accessibilityLabel 전체 컴포넌트 적용 |
